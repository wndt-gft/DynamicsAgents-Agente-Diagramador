Contexto de negócio:
- O banco opera o Sistema de Transferências PIX como plataforma central para pagamentos instantâneos.
- Os canais Mobile Banking App e Internet Banking precisam expor uma jornada unificada para transferências imediatas.
- A régua operacional exige validação de saldo, checagem antifraude, notificações transacionais e geração de comprovantes digitais.
- Todos os eventos devem respeitar regras do BACEN e integrar-se com SPI, DICT e conectores internos do core banking.

História de usuário:
Como gerente de produto responsável pelos canais digitais do Sistema de Transferências PIX,
quero que um cliente PF autenticado possa iniciar uma transferência instantânea pelos canais mobile ou internet,
validando limites, saldo e chaves PIX em tempo real, com dupla checagem antifraude e
notificação imediata do envio e da compensação, para garantir uma experiência confiável e rastreável.

Detalhes e critérios de aceite:
1. O Mobile Banking App e o Internet Banking devem compartilhar os mesmos componentes de backend expostos pelo Serviço de Transferências PIX.
2. Antes da ordem ser enviada ao SPI BACEN, o fluxo deve consultar o Serviço de Validação de Saldo, aplicar o Motor Antifraude PIX e
   gravar o registro da transação no PostgreSQL Transacional PIX e em tópicos Kafka para auditoria.
3. O Serviço de Chaves PIX precisa consultar o DICT BACEN para confirmar a titularidade e acionar o Serviço de Autenticação Transacional
   para validar o segundo fator no canal utilizado.
4. Após a confirmação do SPI, o Serviço de Notificações PIX publica a mensagem via Gateway de Notificações Push e integra com o Conector Core Banking
   para atualizar limites e extratos. As notificações assíncronas também devem estar disponíveis via Redis Cache PIX.
5. O Serviço de Comprovantes Digitais gera PDFs acessíveis nos canais e envia o resumo da transação para o Serviço de Auditoria PIX.
6. Todas as integrações externas (SPI, DICT, Bureau SPC/Serasa) precisam ser representadas como componentes dedicados, respeitando latências máximas de 2 segundos.
7. A arquitetura deve manter isolamento entre as zonas de API Gateway PIX (inbound) e Load Balancer PIX garantindo alta disponibilidade.
8. Exibir na visão de container os componentes tecnológicos principais, suas integrações e fluxos de dados, destacando o core "Sistema de Transferências PIX".
